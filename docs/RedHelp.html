Professional Language Assistant for Construction Evaluation, Notation & Trade Assignment

Version 1.7.46  |  Full Inspection Systems  |  Updated April 2026

Quick Start

  1. Click 📂 Open INS (or press Alt+O) to load an inspection file
  2. Select a checklist item from the left panel
  3. Set the Status (Pass / Fail / NI / Yes / No — depends on item type)
  4. Select the responsible Trade
  5. Add photos and comments as needed
  6. Click ✓ Apply or press Enter
  7. Click 💾 Save when done, or RED auto-saves when you close
💡 Sunlight tip: Selected rows in the inspection picker use high-contrast inverse colors (black background / white text) so they're easy to see in bright sunlight.

Opening the Inspection Picker

Click 📂 Open INS (or Alt+O) to open the Inspection Picker — a scrollable list of inspection files in your MyList folder.

Picker Features

  • Sort by column — click any header to sort ascending/descending (arrow shows direction)
  • Search — type in the search box to filter by address, type, builder, or filename
  • Browse Folder — click 📁 Browse... to load files from a different folder
  • Double-tap or Enter to open the selected inspection
  • Selected row shows black background / white text on all columns — high contrast, easy to see in bright sunlight
  • Swipe to scroll — drag the list with your finger or stylus

Notify Builder (QR Code)

Tap the 📞 button next to any inspection to send a pre-written arrival notice to the builder's phone — without typing on your phone.

  1. Tap 📞 — a window appears with a QR code and a message preview
  2. Point your phone camera at the QR code — your SMS app opens with the message already written
  3. Tap Send on your phone — done

The auto-generated message includes the contact's name, property address, lot number, inspection type, inspector name, and an estimated 20-minute arrival notice.

The 📞 button only appears when a contact phone number is on file for the inspection.

Header Area

At the top of the left panel you'll find:

  • "Insp Items" label showing item totals and your fail count vs. team average (e.g., 12 / 18.5)
  • Trip #N badge — when the filename shows this is a return visit (e.g., Trip #2), this badge appears in the header
  • MegaStats button — a clickable button labeled "N stats" (see MegaStats section below)

Checklist Tree

  • Color coding of each item row tells you its status at a glance (see Status Color table below)
  • Section counters show your fail count / team average for each section (e.g., 3 / 5.2)
  • Per-item stats icons — ⭐ 😃 ↓ ↓↓ ↓↓↓ appear next to items where your rate differs significantly from the team
  • Tap or click any item to open its editor on the right

Bottom Controls

  • ◀ ▶ section arrows — jump to the previous or next section (wraps around)
  • A↓ / A↑ — decrease or increase the checklist font size (saved automatically)
  • 💬 Messages — re-displays any inspection rules or notes from when the file was loaded
  • 📄 See All PDFs (N) — opens a document viewer listing all associated PDFs for this job number
  • Plan button (Slab Eng / EC Plan / Eng Plan / etc.) — opens the primary engineering or plan document; label changes based on inspection type
  • ◀ Trip N (Prior Report) — appears when a prior visit to this property exists; tap to open that earlier inspection for reference

The top section of the editor shows the STATUS controls, which vary by item type:

🗑️ Clear Value Button (wastebasket)

Whenever a value is present on any item, a small 🗑️ wastebasket button appears to the right of the value area. Tap it to instantly clear the value so you can re-enter it or let the EC / Plan data fill it in.

The wastebasket appears for any item that has a value — Pass/Fail buttons, text fields, lookup items, and number pad items alike.

NI Button

For items that support it, the NI button (Not Inspected) appears next to the text box. Tap it to record that this item was not inspected.

Trade / Prefix Area

The TRADE [required] section shows trade buttons (Framing, Electrical, Plumbing, HVAC, etc.). Select exactly one trade — it becomes the prefix for the comment. Tap + to add a custom trade name that persists across sessions.

Comment Area

  • Type your observation in the multi-line text box
  • The preview panel (live) shows what the final comment will look like with prefix and suffix
  • Tap 🗑️ Clear (Alt+C) to erase the comment
  • Tap ✓ Apply (Alt+Y or Enter) to save the comment to the item

Photo Area

  • 📁 File (Alt+F) — import a photo from files
  • 📷 Camera (Alt+M) — capture directly from the device camera; shows "📷 3 photo(s)" when photos already exist
  • 🗑️ Delete (Alt+L) — remove the currently displayed photo
  • ◀ ▶ arrows navigate between multiple photos; counter shows "Photo 1 of 3"
  • Tap any photo to open the Image Editor (see below)
💡 Multi-window camera: When you have two RED windows open and switch windows, the camera in the first window automatically closes. Re-tap Camera in the new window to capture photos for that inspection only.

Location / Suffix Area

Optional multi-select location tags (Kitchen, Master Bath, Garage, etc.) are added as a suffix to the comment. Tap multiple to select all that apply. Tap 📅 to append the current date/time. Tap + to add a custom location.

My Saved Comments

The 📁 MY SAVED COMMENTS section shows comments you've previously saved for this item type. Tap any saved comment to load it. Tap + Save Current (Alt+V) to add the current comment to your library.

Image Editor

Tap any photo in the editor to open it fullscreen for adjustment and annotation:

  • Brightness/Lift — drag up to brighten, drag down to darken
  • Sharpness — drag left/right to adjust sharpness
  • Pen Markup — toggle pen mode to draw arrows, circles, or notes directly on the image
  • Undo — reset all changes back to the original
  • Save — apply edits and return to the main view
  • Cancel (or Escape) — discard changes
💡 Note: Photos are automatically brightened (+25%) when first captured or imported. Sharpening has been removed for faster processing.

AI Features

Get 3 AI Suggestions (Alt+G)

With a photo attached, tap Get 3 to have AI analyze the image and generate three professional inspection comment suggestions. Each suggestion uses standard inspection terminology. Tap any suggestion to insert it into the comment box, or press 1, 2, 3 on the keyboard.

Quick Suggestions (Auto-Overlay)

When the comments box is empty and you navigate to an item, a suggestion overlay automatically appears showing the most common comments written by the team for this specific item (sorted by frequency). Press 1, 2, or 3 to instantly insert, or Escape to dismiss.

Transcribe — Read Labels into the Value Field (Alt+N)

On energy-related inspections (IER, HER, QIER, HET, IEF, HEF, IAP, AFI), a Transcribe button appears in the AI section. Tap it to have AI read text from the photo and place it directly into the Value field — perfect for reading equipment labels, nameplates, and R-value stickers without manual typing.

Great for:

  • Equipment model and serial numbers
  • HVAC capacity and efficiency ratings (SEER, tonnage)
  • Water heater labels (gallon capacity, fuel type)
  • Insulation R-value stickers
  • Window U-factor and SHGC stickers
  • Any nameplate with data you need to record

Auto-Fill Adjacent Items

When you transcribe a label that contains multiple values (for example, a window sticker showing "U-Factor: 0.28 / SHGC: 0.22"), RED automatically fills the value in the current item and checks adjacent items to fill related values. For example, transcribing on item 2.2 (U-factor) may also populate item 2.3 (SHGC) if both values appear on the same label.

Tips for best results:

  • 📸 Hold the camera so text is upright and level — not tilted or at an angle
  • Get close enough that the text is clearly legible
  • Avoid glare on shiny labels — tilt slightly to reduce reflections
  • Make sure the full label is in frame
  • Good lighting helps — use your phone flashlight in dark spaces
💡 Tip: After Transcribe fills a value, do a quick sanity check. AI is very accurate but not perfect — one glance catches the occasional misread digit.

Energy Compliance (EC) Banner NEW since v1.4

On energy inspections, a banner appears above the item editor that shows design values extracted from the Energy Compliance (EC) report PDF. This lets you instantly compare what was designed against what you observed in the field.

Which Inspection Types Show the EC Banner

CodeInspection Name
IER / HER / QIEREnergy Star Rough (insulation)
IEF / HEFEnergy Star Final (floor / equipment)
HETEnergy Star Final Testing (blower door / duct leakage)
IAPIndoor Air Plus
AFIACCA 310 Field Inspection
PLY / PS / ACIPolyseal / Performance path inspections

Banner Colors

The banner changes color based on how the measured (field) value compares to the design value from the EC report:

  • ✔ Green — Measured value matches design requirement
  • ✗ Red — MISMATCH — Value is blank when one is required, or is out of acceptable range
  • Gray — No data — No design value mapped to this item
  • ⚠ Amber — EC report found but data could not be fully extracted (tap for details)

Red Flash / Strobe

When the banner transitions to red for the first time on an item (mismatch detected), it strobes red/white five times to catch your attention before settling red. This strobe only fires when switching from non-red to red — if the banner was already red when you navigated to the item, it does not strobe again.

What the Banner Shows

The banner displays data chips for the fields relevant to the current item:

  • U-factor: design value from EC report (e.g., "U-factor: 0.28")
  • SHGC: design solar heat gain coefficient (e.g., "SHGC: 0.22")
  • R-values: ceiling, wall, duct, pipe, attic
  • HERS Index: overall home energy rating score
  • Blower Door: maximum allowed air leakage in CFM
  • Duct Leakage: maximum allowed duct leakage in CFM
  • Fresh Air: minimum required ventilation CFM
  • SEER: minimum required AC efficiency
  • Floor Area / Volume / Bedrooms: for final energy form items

Comparison Logic

Item TypeComparisonWhy
U-factor (IER 2.2)Must match design spec (±5%)Installed windows must match the energy model
SHGC (IER 2.3)Must match design spec (±5%)Installed windows must match the energy model
R-values (insulation)Actual ≥ design minimumMore insulation than designed is acceptable
Blower Door, Duct LeakageActual ≤ design maximumLower leakage than designed is acceptable
SEER, Fresh AirActual ≥ design minimumBetter efficiency / more ventilation is acceptable
Floor Area, BedroomsClose match (within 5%)Must match the home that was modeled
Energy Star Program, WH FuelText contains design valueMust match program type / fuel type
💡 U-factor & SHGC: These use a close-match comparison (within 5% of the design value). If your field value is 0.20 and the design says 0.22, that flags red — even though 0.20 is technically "better" energy-wise, the installed window doesn't match what was modeled. Check with the builder if different windows were substituted.

⬇ Apply EC Button

When the EC banner is green and the current item has no value yet, the ⬇ Apply EC button appears. Tap it to instantly fill the current item's value from the EC report. If the item already has a value, the button is grayed out (non-destructive — it won't overwrite your work).

Tapping the Banner

Tap anywhere on the EC banner to open a detail overlay showing every data point extracted from the EC report — HERS index, all R-values, duct specs, blower door, floor area, IECC version, and more. Useful for cross-checking multiple items at once.

Slab / Structural Banner (CPP Inspections) NEW since v1.4

On CPP (Concrete Pre-Pour) inspections, a separate banner shows values extracted from the foundation detail sheet or other structural plan PDF. This allows you to compare your field observations against the engineering plan without switching documents.

What It Extracts

  • Cable count — total post-tension cables, split into F2B (front-to-back) and R2L (right-to-left) directions
  • Slab thickness — design thickness in inches
  • Beam dimensions — width × depth
  • Holddown count — seismic anchor quantity

Cable Count Comparison

Cable counts are the most critical slab measurement. RED compares your recorded count to the plan:

  • Green — count matches plan (within ±1 cable per direction)
  • Red — MISMATCH — count is wrong (triggers strobe flash)

⬇ Apply Plan Button

When slab data is available and the current item has no value, the ⬇ Apply Plan button fills the item automatically. For item 5.1.b (cable count), it also writes a note in the comments field with the directional breakdown.

Tapping the Slab Banner

Tap the banner to see the full extracted slab data — cable counts, slab thickness, beam specs, holddown count, and the PDF it was read from. A link to the raw OCR text is also provided for troubleshooting.

Plan PDF Revision Selection (CPP)

When multiple plan revisions exist for a job number, right-click the Slab Eng button to select which revision to use. The most current revision is selected automatically by default.

MegaStats Panel NEW since v1.4

The MegaStats button in the left panel header (labeled "N stats") opens an overlay showing the items most commonly found by the entire team on this inspection type — giving you a quick reference for what to focus on during your inspection.

What You See

For each high-frequency item, the panel shows:

  • The item number and name
  • A bar chart showing the team catch rate (how often this item is flagged as a fail)
  • Your personal catch rate vs. the team rate — so you can see where you're strong and where you might have a blind spot
  • Quick comments — the most common comment written for this item across all inspectors

Notes

  • Section 1 (administrative) items are excluded — those are checkoffs, not real inspection findings
  • Only items with a team catch rate of 3% or higher appear (items rarely caught aren't shown)
  • Up to 15 items are shown, sorted by team catch rate (highest first)
  • Swipe up/down within the overlay to scroll through all items
  • Tap the ✕ button or the dark backdrop to close the panel
💡 Tip: Open MegaStats at the start of an inspection as a quick reminder of what the team consistently finds on this form type. It's especially helpful for inspection types you don't do every day.

Trip Number & Prior Reports NEW since v1.4

Trip Number Badge

When an inspection file name indicates this is a return visit (e.g., trip 2 or trip 3), a Trip #N badge appears in the left panel header. This is a quick visual reminder that you're on a re-inspection.

Prior Report Button

When a previous trip to the same property exists in your MyList folder, a ◀ Trip N button appears at the bottom of the left panel. Tap it to open that earlier inspection in a second RED window for side-by-side comparison — useful for verifying that items that failed on the prior visit have been corrected.

Inspector Statistics

Understanding Your Stats

RED tracks your fail counts across all your inspections and compares them to the team average — helping you identify blind spots and see where you're thorough.

Section-Level Counters

Each section header shows a counter like (3 / 5.2):

  • First number (3) — your current fail count for this section
  • Second number (5.2) — the team average fails per inspection for this section

Inspection-Level Counter

At the top next to "Insp Items" you'll see a total like (12 / 18.5) — your total fails vs. the team average for the whole inspection.

Per-Item Indicators

IndicatorMeaning
⭐100%+ above team average — exceptional attention to detail
😃10%+ above team average — above average
↓40–59% below team average — worth a closer look
↓↓60–79% below team average — blind spot
↓↓↓80%+ below team average — significant blind spot

Stats Scope (Global / Builder / Neighborhood)

The title bar shows three clickable mode labels. Tap one to change the comparison scope:

ModeCompares Against
GlobalAll inspections, all builders — full team average
Builder ClientOnly inspections for the same builder (e.g., DR Horton, Pulte)
NeighborhoodOnly inspections in the same subdivision — hyper-local comparison

The active mode shows in white text with an underline. Section counters and item indicators update immediately. If there isn't enough data for Builder or Neighborhood mode, RED falls back to Global automatically.

Required Field Indicators

RED uses color to highlight missing required fields so nothing gets overlooked:

ColorMeaning
Saturated Red backgroundRequired field is MISSING (no value or no photo yet)
Light Blue backgroundField is filled, or not required

Both the VALUE section and the PHOTO section show a red background when their required content is missing. The moment you supply the value or photo, the background changes to blue.

Templates

Creating a Template (Alt+T)

  1. Open an inspection and set your preferred default values for each item
  2. Click Save Template or press Alt+T
  3. Enter a name (e.g., "Standard Pre-Pour", "Reinspection Defaults")
  4. The template saves the status values only — not comments or photos

Applying a Template (Alt+A)

  1. Open a new inspection file
  2. If a matching template exists, you'll be prompted automatically on load
  3. Or click Apply Template / press Alt+A at any time
  4. Select from your saved templates for this form type
  5. All status values are applied instantly
💡 Tips: Templates are per form type — different inspection types have separate template libraries. Create multiple templates for different scenarios (e.g., "New Construction" vs. "Reinspection"). Stored locally in %LOCALAPPDATA%\Red\templates.json.

Fail-to-Pass Smart Cleanup

When you change an item from Fail → Pass (common on re-inspections), RED asks:

"Do you want to remove picture and comment as well?"
[Yes]   [No — leave comment and image]
  • Yes: clears the comment and photos — unless the item requires a photo, in which case the required photo is protected and stays attached
  • No: nothing is removed — comment and photos stay in place
💡 Tip: When a photo is required, RED shows a warning: "⚠️ Picture is REQUIRED and will NOT be deleted." You can safely click Yes knowing the required photo remains.

Multiple Inspections (Multi-Window)

Opening a Second Inspection Simultaneously

  1. With an inspection open, click 📂 Open INS
  2. A dialog asks:
    • Yes — keep this one open and launch a new RED window
    • No — close this one and open the new file here
    • Cancel — never mind
RuleWhy
❌ Cannot open the same file in two windowsPrevents conflicting edits to the same report
❌ Cannot open two inspections of the same type simultaneouslyStats and templates are type-specific — duplicate types would cause confusion
✅ Can open different types side by side (e.g., IER + CPP)Each window tracks its own type independently
⚠️ Remember: RED auto-saves when you close a window — but if you have two windows open, you must save/close each one separately.

Layout & Customization

Resizing Panels

A draggable divider sits between the left checklist panel and the right editor panel. Drag it left or right to adjust widths. Your preferred split is saved and restored automatically.

Checklist Font Size

Use A↓ and A↑ at the bottom-left to decrease or increase the checklist text size. Your preference is saved automatically.

Section Navigation

The ◀ and ▶ buttons at the bottom-left jump between major form sections without scrolling. Wraps around (last → first, first → last).

Item Duplication

Some defects need to be recorded multiple times (e.g., the same framing issue in three locations). Use ➕ Duplicate (Alt+U) to clone the current item. The duplicate appears in the same section and can have its own value, comment, photo, and location tags. Use ➖ Remove (Alt+R) to delete a duplicate (originals cannot be removed).

Efficient Workflow Tips

Speed Run Method

  1. Open inspection with Alt+O
  2. Apply template if prompted
  3. Check MegaStats panel to see what the team commonly finds on this form
  4. Use INC filter to see which required fields still need values
  5. Use ↓ to move through items one by one
  6. For each defect: Tab to Status → Space on Fail → Tab to Trade → Space → Tab to Comments → type or use suggestions → Enter
  7. Save with Alt+S

Energy Inspections — EC Banner Workflow

  1. Open the IER / HET / IEF file
  2. EC data loads automatically in the background
  3. Navigate to items 2.2 (U-factor) and 2.3 (SHGC) — tap ⬇ Apply EC if the fields are empty and the EC data matched
  4. For insulation items, check the banner color — green means it matches the design, red means it doesn't
  5. If a banner flashes red, record the actual measured value and note the discrepancy in the comment

CPP Slab Inspection Workflow

  1. Open CPP file — the slab banner loads automatically
  2. Count F2B and R2L cables on the slab
  3. Navigate to the cable count items — the banner shows the design count
  4. If counts match, tap ⬇ Apply Plan to fill the value; banner turns green
  5. If counts differ, record your actual count — banner turns red (and strobes) as a flag

Use OFI for Review

Before saving, click OFI to see only your failures and do a final review of each one — comments, photos, trades all correct.

Use INC + MegaStats Together

Open MegaStats to see the most commonly flagged items, then switch to INC to make sure all required fields are filled before you leave the site.
